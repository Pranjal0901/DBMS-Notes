Basic Terminologies of DBMS

Relation:It refers to a table that stores data. It's a way to organize information into rows and columns.In simpler terms, a relation is basically a structured way of keeping track of       information.

Tuple: It refers to a single row of data in a row.Each row in the table represents a unique record or set of data, and a tuple is essentially one of these rows.

Attributes:Attributes are the characteristics or properties that describe the entities (objects or things) you are storing information about.

Primary Key:It is like a special ID or label for each row in a table. It's a unique identifier that helps distinguish one row from another.

Degree:It refers to the number of attributes or columns in a relation (table). It tells how many pieces of information or characteristics are stored for each entry (tuple) in the table.

Cardinality:It refers to the relationship between tables, specifically the number of rows (records) in one table that can be related to the number of rows in another table

Database System Architecture and its levels

Database Architecture

 It refers to the design and structure of databse and shows how the componnets of the system interact and work together. It is important for understanding how data is prcessed,  stored and access in the system

 Types of DBMS Architecture

 There are three types of DBMS Architecture:

Single Tier Architecture

Two Tier Architecture

Three Tier Architecture

Single tier Architecture

In this type of Architecture are commonly found in desktop databse application where databse engine and user interface are bundled together. Any request made by client it doesn't require network connection.

Advantages of Single Tier Architecture

Single-tier architecture is simple to set up and maintain, as all components are on a       single machine.
Single-tier architecture can be deployed quickly and easily.

Two tier Architecture

In two-tier architecture, the database system is present at the server machine and the DBMS application is present at the client machine, these two machines are connected with each other through a reliable network.

Advantages of Two Tier Architecture

The client has direct access to the database, allowing for fast data retrieval and     update.
Some processing can be done on the client side, reducing the load on the server     and improving performance.

Three tier Architecture

In three tier architecture, the client application doesn't communicate directly with the database system present at the server machine, rather the client apllication communicates the server application and the server application internally communicates with the databse system present at the server.

Advantages of Three Tier Architecture

The architecture can be easily scaled by adding more servers or upgrading existing       servers, improving performance, and ensuring high availability.
The three-tier architecture provides easier to secure data and control access to the       database.

Three-level ANSI SPARC Database Architecture

The ANSI SPARC(American National Standard Institute Standards Planning and Requirements Committee)three level databse architecture is a conceptual framework that was intoducedto standardize the design and development of database management system(DBMS).

It has main three levels:

Internal Level

Conceptual Level

External Level

Internal Level

It is also known as physical level
It is the lowet level of data abstraction.
It represents the way data is physically stored adn accessed in     storage.
It deals with complex low level data structures, file structres     and access methods in detail.

Conceptual Level

It is also known as Logical level.
It is the higher level than the internal level of     data abstraction.
It hides low level complexitites of physical     storage.
The conceptual schema defines the structure     of the entire database.

External Level

It is also known as view level.
It is the highest level of data abstraction.
End users need to access only part of the database     rather than the entire databse.
Different users need sofferent views of the datbase

Advantages of Three Tier Architecture

It provide Data Abstraction.
Same data can be accessed by different users with different customized views.
Physcial storage structure can be chaged without requiring change in internal sturcture.

DBA(Database Administrator)

DBA stand for Database Administrator.A DBA is a professional responsible for the management, maintainance and optimization of a database system. The DBA makes sure that database is protected and that any chance of data loss is minimized.It plays a crucial role in ensuring the reliability, security, and performance of database.

Responsibilities of DBA

The responsibilities of the database administrator are:-

Installing and Upgrading an SQL Server

The DBA is responsible for installing SQL Server or upgrading an existing SQL server.It is also responsible for applying SQL Server Service packs.

Monitoring the Database Server's Health and Tuning Accordingly

It can be done by making sure that the followoing is done:

The Server is running with optimal peformance

The error or event log is monitored for databse errors

Database have routine maintainance performed on them by system administrator.

Using Storage Properly

SQL Server enables you to automatically grow the size of your databse and transaction log,or you can select a fixed size for the databse and transaction log.

Performing Backup and Recovery Duties

Backup And Recovery are the DBA's most critical task.

Establishing standards adn schedules for databse backups.

Developing recovery procedured for each database.

Makes sure that the backup schedulesmeet the recovery requirements.

Managing Database Users and Security

The DBA works tightly with Windows NT administrator to add user NT login to database.The DBA is responsible for assigning users to databses and determinig the proper security level for each user.

Working with Developers

It is important for the DBA to work closely with development teams to assist in overall databse design, such as creating database, helping in queries etc.

Establishing and Enforcing Standards

The DBA should establish naming conventions and standards for the SQL Server and databases and make sure that everyone sticks to them.

Transferring Data

The DBA is responsible for importing and exporting data to and from the SQL server.

Replicating Data

SQL Server version 2000 has many different replication capabilities such as merge replication and queued replication.

Data Warehousing

SQL server has substatial data Warehousing capabilities that require the DBA to learn an additional product(Microsoft OLAP Server) and architecture

Scheduling Events

The DBA is responsible for setting up and scheduling events using SQL server to perform many taask such as replication and backup.

Providing 24-Hour Access

The Database server must stay up, and the databse must always e protected and online.Be prepared to perform the maintainanceand upgrades.

Different Tpyes of Database Users:-

There are seven types of database users in DBMS:-

Database Administrator(DBA)

It is a person who defines the schema and also controls the three levels of database.
It is responsible for providing security to databse and allow only authorized users.
It monitors the recoveryand backup and provide technocal support.

Naive/Parametric End Users

Those users who don't have any any DBMS knowledge but they frequently use database applications in their daily life to get result. Ex- Railway Ticket booking users.

System Analyst

System Analyst is a user who analyzes the requirements of parametric end users. They check whether all the requirements of end users are satisfied.

Sophisticated Users

Those users who are familiar with the database.They develop their own database applications according to their requirement. They directly interact the database by using SQL Queries.

Database Designers

The users who design the structure of database which includes tables, indexes, views, constraints, triggers.They control what data must be stored and how the data items to be related.

Application Program

Application Program are the back-end programmers who writes the code for the application program.It can be written in C, COBOL, FORTRAN language etc.

Casual Users/Temporary users

The users who occasionally use the database but each time they access, they require the new information. Ex- Middle or High level Manager.
